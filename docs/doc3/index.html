Passing data between components and templates

Check the documentation for Angular Framework.

Docs

Angular applications are mainly formed of components and modules. Components are parts of an application and they can be a button, menu, a simple text with a tooltip or even a whole page. Modules have the purpose to group other components, pipes, directives or services that are related, which together create a part or even a whole application. We can create new components and modules with commands.

In components you can display data by binding controls in an HTML template to properties of an Angular component. The easiest way to display a component property is to bind the property name through interpolation. With interpolation, you put the property name in the view template, enclosed in double curly braces: {{myJoke}}.

🏋️ Exercise time

Communication between components can be done through Inputs and Outputs. Each component has a life-cycle. There are multiple hooks which can be used for multiple purposes: OnInit, OnChange, OnDestroy

Examples

Creating a component

ng g c joke-card
g -> generate
c -> component

@Component({
  selector: 'app-chuck',
  templateUrl: './chuck.component.html',
  styleUrls: ['./chuck.component.scss']
})
export class ChuckComponent implements OnInit {

  constructor() { }

  ngOnInit() {
  }

}

Creating a module

ng g m chuck
m -> module
